# 智能合约审计合规指南

## 一、安全红线（必须检查）

### 1. 资金安全类
- [ ] 未经授权的资金转移
- [ ] 重入攻击漏洞
- [ ] 整数溢出/下溢
- [ ] 前置交易（Front-running）风险
- [ ] 价格操纵漏洞

### 2. 访问控制类
- [ ] 权限提升漏洞
- [ ] 缺失 Owner 检查
- [ ] 函数访问控制不当
- [ ] 初始化函数可重复调用

### 3. 逻辑缺陷类
- [ ] 竞争条件
- [ ] 时间戳依赖
- [ ] 除零错误
- [ ] 死锁风险

## 三、法律法规参考

### 1. 证券法合规（如适用）
- Howey 测试四要素
- 去中心化程度评估
- 投资合同特征识别

### 2. 数据隐私
- GDPR 合规（如处理个人数据）
- 链上数据不可删除性考虑

### 3. 金融监管
- DeFi 协议监管要求
- KYC/AML 合规性

## 四、审计责任声明

1. 审计报告仅供参考，不构成投资建议
2. 审计不保证合约 100% 安全
3. 建议进行多重审计和漏洞赏金计划
4. 审计范围需明确定义
